Omitting columns does not omit Record data

Columns do not need to be included in DataTable's column definition for the data that was captured for its RecordSet to include the fields identified in the DataSource schema's fields configuration.

In this example, the DataSource captures three columns of data, but only one column is included in the DataTable's column definition. A formatter is added to this column to show how to access other field data from the record, regardless of whether those fields have corresponding columns.
